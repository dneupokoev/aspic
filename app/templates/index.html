🔗 Webhook URL
🔗 Webhook URL

Ссылка на ваш внешний API для контроля доступа к файлу.

🎯 Зачем нужно:
  • Ограничить доступ только для определённых пользователей
  • Привязать файл к номеру заказа, ID клиента или сессии
  • Интегрировать с вашей системой авторизации
  • Отслеживать попытки доступа к файлу
🔍 Как это работает:
  1. При каждой попытке открыть файл ASPIC отправляет GET-запрос на ваш URL
  2. В запрос автоматически добавляются все параметры из ссылки (например, ?order_id=123&user=john)
  3. Ваш API должен вернуть JSON: {"allowed": "1"} - доступ разрешён, {"allowed": "0"} - доступ запрещён
📌 Пример:

Если вы укажете https://api.example.com/check, а ссылка будет /view/abc123?order_id=INV-001, то ASPIC вызовет:

https://api.example.com/check?token=abc123&order_id=INV-001&client_ip=1.2.3.4
⚠️ Важно: URL должен начинаться с http:// или https://

🔒 Пароль для удаления
🔒 Пароль для удаления
🎯 Зачем нужно:
  • Защитить файл от случайного или злонамеренного удаления
  • Только тот, кто знает пароль, сможет удалить файл
  • Остальные смогут только просматривать и комментировать
🔍 Как это работает:
  1. Вы устанавливаете пароль при загрузке файла
  2. При попытке удаления сайт запросит этот пароль
  3. Без правильного пароля удалить файл невозможно
📌 Требования: Длина от 4 до 16 символов. Можно использовать буквы, цифры и спецсимволы.
⚠️ Внимание: Используйте только для базовой защиты, не для критически важных данных.

⏰ Автоматическое удаление
⏰ Автоматическое удаление

Вы можете настроить автоматическое удаление файла одним из способов:

  • Конкретная дата - файл будет удалён в указанный день в 00:00
  • Часов после последнего обращения - файл будет удалён через указанное время после последнего просмотра или скачивания

Если оба поля заполнены - файл будет удалён по тому условию, которое наступит раньше.

📌 Пример: Укажите "24" - файл будет удалён через 24 часа после того, как его в последний раз открыли или скачали.
📌 Значения по умолчанию: дата удаления = {{ default_expire_date }}, TTL = {{ format_ttl_hours(default_ttl_hours) }}
