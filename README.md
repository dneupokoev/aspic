# 📦 ASPIC - A Simple Public Image/File Cloud

**ASPIC** - это минималистичный файловый хостинг с возможностью внешней (через вебхук) проверки доступа к загруженному файлу! Дополнительно имеется: комментирование, безопасное удаление файлов, автоматическая очистка.

Но самая главная функция - это **проверка доступа к файлу через webhook**.

**Зачем это нужно?**

У меня была такая система: пользователи писали в бот мессенджера и скидывали в него файлы, я эти файлы сохранял на диске с доступом наружу (в интернет).
Далее менеджеры просматривали эти файлы и отвечали пользователям. Всё это происходит на сайте, где по F12 можно посмотреть исходник и получить ссылку на файл (прямо как в Максе), но **требуется ограничить доступ** к этому файлу.
Для этого и нужен вэбхук, который проверит и решит отображать файл или нет - то есть ссылка на файл откроет файл только если разрешит вэбхук.

Ещё пример: Вы загружаете коммерческое предложение для конкретного партнёра. Ссылка утекает к конкуренту - и всё, ваша цена и условия стали публичными. Что может проверять вэбхук: убедиться, что запрос пришёл с адреса его личного кабинета или с определенного IP.

Общая суть: вэбхук-проверка нужна везде, где:
- ссылка на файл не должна быть вечной и публичной;
- доступ должен зависеть от контекста (кто, когда, откуда, с какими правами);
- вы не хотите доверять безопасность только тому, что ссылка «длинная и случайная».

ASPIC в этих сценариях - это не просто хостинг, а контроллер доступа перед тем, как файл улетит в браузер.

**ASPIC решает** эту главную для меня задачу! А раз уж проект был выделен в отдельный, я добавил ему "красивый" интерфейс и дополнительные возможности...

## ✨ Возможности

### 📤 Загрузка файлов
- ✅ **Три способа загрузки**: drag-and-drop, выбор файла или вставка из буфера (Ctrl+V)
- ✅ **Редактирование текста** перед созданием ссылки при вставке из буфера
- ✅ **Принимаются любые файлы** - никаких ограничений по типу
- ✅ **Предпросмотр** перед загрузкой с информацией о файле
- ✅ **Автоматическое определение MIME-типа** по содержимому
- ✅ **Обход лимита размера** через секретный параметр (до 4 ГБ) с безопасным хэш-токеном
- ✅ **Прямая загрузка через API** (`POST /api/upload`) с авторизацией по токену
- ✅ **Автоматическая структура папок** по месяцам (YYYY/MM)
- ✅ **Проверка свободного места** на диске перед загрузкой (настраиваемый порог)
- ✅ **Ограничения для обычных пользователей**: лимит файлов и суммарного объёма за период
- ✅ **Отображение остатков лимитов** на главной странице в реальном времени

### 🔗 Ссылки и доступ
- ✅ Получение **вечной ссылки** на файл (живёт, пока кто-то не удалит или не сработает автоудаление)
- ✅ **Две ссылки** на файл: страница просмотра и прямая ссылка для встраивания
- ✅ **Предпросмотр** изображений, PDF, видео, аудио и текста прямо в браузере
- ✅ **Статистика**: отдельные счетчики просмотров и скачиваний с датами последних действий
- ✅ **Защита от накрутки**: уникальные просмотры с одного IP раз в час

### ⏰ Автоматическое удаление файлов
- ✅ **Настройка при загрузке**: возможность указать конкретную дату удаления или TTL (часов после последнего обращения)
- ✅ **Гибкие правила**: если указаны оба параметра - файл удаляется по первому наступившему условию
- ✅ **Отображение на странице файла**: информация о сроках жизни файла видна всем посетителям
- ✅ **Автоматическое удаление по истёкшей дате**: при загрузке нового файла сервер проверяет и удаляет файлы с истёкшей датой
- ✅ **Защита от частых запусков**: автоудаление срабатывает не чаще раза в N часов (настраивается в `.env`)

### 🔐 Внешняя авторизация через Webhook
- ✅ Возможность указать **URL внешнего API** для проверки доступа к файлу
- ✅ При открытии файла ASPIC передаёт **все параметры запроса** в вебхук
- ✅ Вебхук должен вернуть `{"allowed": "1"}` или `{"allowed": "0"}`
- ✅ **Кэширование ответов** (5 минут) для одинаковых параметров
- ✅ **Rate limiting**: защита от злоупотреблений на уровне IP и URL вебхука
- ✅ **Таймауты**: на подключение (3с), чтение (5с) и общий (5с)
- ✅ **SSRF защита**: блокировка внутренних IP и локальных адресов
- ✅ **Ограничение размера ответа**: максимум 1 МБ
- ✅ **Отключены редиректы** для предотвращения использования как прокси

### 💬 Комментарии и взаимодействие
- ✅ **Комментарии** к файлам с автоматическим сохранением имени автора
- ✅ **Удаление файлов** с подтверждением через капчу и указанием причины
- ✅ **Защита паролем** - возможность установить пароль для удаления файла при загрузке
- ✅ При удалении комментарий с причиной сохраняется в истории

### 🛡️ Безопасность
- ✅ **Rate limiting** для всех действий (загрузка, комментарии, удаление)
- ✅ **Капча**: обязательна для удаления файлов (защита от ботов)
- ✅ **Парольная защита**: опциональный пароль для удаления (от 4 до 16 символов)
- ✅ **Валидация MIME-типов**: определение по содержимому, а не по расширению (защита от подделки)
- ✅ **Soft delete**: файлы помечаются удаленными, но физически удаляются отдельным скриптом
- ✅ **Изоляция файлов**: автоматическая структура папок по месяцам (YYYY/MM)
- ✅ **Защита от Path Traversal**: проверка имен файлов при загрузке
- ✅ **Безопасный обход лимита**: хэш-токен в куке, срок жизни 1 час, защита от подделки
- ✅ **HTTP-only куки** для user_id
- ✅ **Проверка свободного места**: защита от заполнения диска
- ✅ **API токены**: авторизация через заголовок `X-Upload-Token` для загрузки через API
- ✅ **Ограничения для обычных пользователей**: лимит количества файлов и суммарного объёма за период (скользящее окно)

### 🎨 Интерфейс
- ✅ **Адаптивный дизайн** для мобильных устройств
- ✅ **Темная тема** (автоматически подстраивается под систему)
- ✅ **Валидация форм** в реальном времени с подсветкой ошибок
- ✅ **Информационные попапы** с пояснениями для всех полей
- ✅ **Копирование ссылок** одним кликом с обратной связью
- ✅ **Возможность отключения веб-интерфейса** загрузки файлов (режим API-only)

### 🛠 Технические особенности
- ✅ Полная изоляция: файлы хранятся на диске, метаданные в SQLite
- ✅ **WAL-режим** SQLite для лучшей производительности при конкурентном доступе
- ✅ **Абстракция хранения**: поддержка локального диска и готовность к S3/MinIO
- ✅ **Фоновые задачи** для очистки временных файлов
- ✅ Гибкая настройка через `.env` с подробными комментариями
- ✅ Готов к запуску в **Docker**
- ✅ **Cleanup-скрипт** для физического удаления помеченных файлов

## 📋 Примеры использования

### Простой файл (без ограничений)
1. Перетащите файл в область загрузки
2. Нажмите «Создать ссылку»
3. Поделитесь ссылкой - файл доступен всем

### Файл с автоматическим удалением
1. Загрузите файл
2. В блоке «Автоматическое удаление» укажите:
   - **Конкретную дату** - файл будет удалён в указанный день в 00:00
   - **Часов после последнего обращения** - файл будет удалён через указанное время после последнего просмотра или скачивания
3. Если указаны оба параметра - файл будет удалён по первому наступившему условию
4. На странице файла будет отображаться информация о сроках жизни

### Файл с защитой через внешний API
1. Загрузите файл
2. В поле «Webhook URL» укажите URL вашего API: `https://api.example.com/check-access`
3. Создайте ссылку
4. Поделитесь ссылкой с параметрами: `https://aspic.example.com/view/abc123?order_id=INV-2025-042&user_id=789`
5. ASPIC вызовет ваш API с параметрами: `https://api.example.com/check-access?token=abc123&order_id=INV-2025-042&user_id=789&client_ip=1.2.3.4`
6. Ваш API возвращает `{"allowed": "1"}` или `{"allowed": "0"}`

### Файл с защитой паролем от удаления
1. Загрузите файл
2. В поле «Пароль для удаления» введите пароль (например, `MySecret123`)
3. Создайте ссылку
4. Теперь для удаления файла потребуется ввести этот пароль

### Загрузка больших файлов (до 4 ГБ)
1. Убедитесь, что в `.env` установлен `UNLIMITED_UPLOAD_SECRET=ваш_секрет`
2. Откройте главную страницу с параметром: `/?upload_secret=ваш_секрет`
3. Сервер установит безопасную куку на 1 час
4. Загрузите файл до 4 ГБ обычным способом
5. Кука автоматически передаётся при загрузке - никаких дополнительных действий

> ⚠️ **Важно**: для загрузки файлов >100 МБ также увеличьте `client_max_body_size` в Nginx

### Режим API-only (только API, без web-интерфейса загрузки)
1. Установите в `.env`: `ENABLE_WEB_UI=False`
2. Перезапустите сервер
3. Главная страница (`/`) теперь возвращает 404
4. Все существующие файлы доступны для просмотра (`/view/токен`)
5. API продолжает работать в полном объеме

## 🛡️ Безопасность вебхуков (для администраторов)

ASPIC включает многоуровневую защиту от злоупотреблений вебхуками:

| Защита | Описание | Настройка в `.env` |
|--------|----------|-------------------|
| **SSRF защита** | Блокировка внутренних IP (localhost, 10.0.0.0/8, 192.168.0.0/16 и др.) | Встроено, не настраивается |
| **Rate limiting по URL** | Не более X вызовов одного вебхука в минуту | `WEBHOOK_RATE_LIMIT_PER_URL=10` |
| **Rate limiting по IP** | Не более X вызовов вебхуков с одного IP в минуту | `WEBHOOK_RATE_LIMIT_PER_IP=30` |
| **Concurrent limit** | Не более X одновременных запросов с одного IP | `WEBHOOK_MAX_CONCURRENT_PER_IP=5` |
| **Таймауты** | Общий (5с), подключение (3с), чтение (5с) | `WEBHOOK_TIMEOUT=5`, `WEBHOOK_CONNECT_TIMEOUT=3`, `WEBHOOK_READ_TIMEOUT=5` |
| **Размер ответа** | Не более 1 МБ | `WEBHOOK_MAX_RESPONSE_SIZE=1048576` |
| **Кэширование** | Кэш ответов на 5 минут для одинаковых параметров | `WEBHOOK_CACHE_TTL=300` |
| **Протоколы** | Только HTTPS в production, HTTP разрешён в DEBUG | Автоматически |

## 🔐 Безопасность обхода лимита размера

Механизм обхода лимита размера файла реализован с учётом безопасности:

| Компонент | Описание |
|-----------|----------|
| **Хэш-токен** | В куке хранится `SHA256(секрет:таймстамп:соль)`, а не сам секрет |
| **Срок жизни** | Кука действительна 1 час (3600 секунд) |
| **Защита от подделки** | Используется `secrets.compare_digest()` для сравнения хэшей |
| **HTTPS в production** | Кука передаётся только по HTTPS (`secure=not DEBUG`) |

> 🔒 **Важно**: без знания `UNLIMITED_UPLOAD_SECRET` из `.env` подделать токен невозможно.

## ⚙️ Настройка

Все настройки в файле `.env` с подробными комментариями. Основные группы параметров:

- **Основные настройки**: `DEBUG`, `HOST`, `PORT`
- **Веб-интерфейс**: `ENABLE_WEB_UI` (включение/отключение загрузки)
- **Пути к данным**: `UPLOAD_DIR`, `PREVIEW_DIR`, `DB_PATH`
- **Rate limiting**: `RATE_LIMIT_UPLOAD`, `RATE_LIMIT_COMMENT`, `RATE_LIMIT_DELETE`
- **Лимиты размера**: `MAX_FILE_SIZE`, `UNLIMITED_UPLOAD_SECRET`, `MAX_UNLIMITED_FILE_SIZE`
- **Дисковое пространство**: `MIN_DISK_SPACE` (минимальный порог свободного места)
- **Автоматическое удаление**: `DEFAULT_EXPIRE_DAYS`, `DEFAULT_TTL_HOURS`
- **Автоудаление по истёкшей дате**: `AUTO_CLEANUP_MIN_INTERVAL_HOURS`
- **API токены**: `API_UPLOAD_TOKENS` (формат `токен:Имя`, через запятую)
- **Ограничения для обычных пользователей**: `UPLOAD_LIMIT_WINDOW_HOURS`, `UPLOAD_LIMIT_MAX_FILES`, `UPLOAD_LIMIT_MAX_FILE_SIZE`, `UPLOAD_LIMIT_MAX_TOTAL_SIZE`
- **Безопасность вебхуков**: таймауты, rate limiting, ограничения размера

## 📏 Ограничения для обычных пользователей

Пользователи, которые **не знают** `UNLIMITED_UPLOAD_SECRET`, подчиняются общим лимитам:

| Параметр | По умолчанию | Описание |
|----------|-------------|----------|
| `UPLOAD_LIMIT_WINDOW_HOURS` | 6 | Скользящее окно (часы) |
| `UPLOAD_LIMIT_MAX_FILES` | 100 | Макс. файлов за окно |
| `UPLOAD_LIMIT_MAX_FILE_SIZE` | 100 МБ | Макс. размер одного файла |
| `UPLOAD_LIMIT_MAX_TOTAL_SIZE` | 500 МБ | Макс. суммарный объём за окно |

- На главной странице **отображаются остатки** в реальном времени: сколько загружено, сколько осталось
- При превышении лимита - **ошибка 429** с подробным описанием и **временем ожидания**
- API пользователи (с `X-Upload-Token`) **не подчиняются** этим лимитам
- Лимиты **общие для всех** обычных пользователей (не по IP)

## 🔑 API токены для загрузки

Эндпоинт `POST /api/upload` требует авторизации через заголовок `X-Upload-Token`.

```env
API_UPLOAD_TOKENS=tok_abc123:Менеджер Иван,tok_def456:API Сервис XYZ
```

- **Формат**: `токен:Имя владельца`, через запятую
- **Пустое значение** = эндпоинт `/api/upload` отключён (вернёт 503)
- Автор загрузки **сохраняется в БД** в поле `author`
- API пользователи загружают файлы **без ограничений** (до `MAX_UNLIMITED_FILE_SIZE`, до 4 ГБ)
- При загрузке можно указать: `webhook_url`, `delete_password`, `expire_date`, `ttl_hours`

## 🚀 Установка

### Через Docker

```bash
# Сборка образа
docker build -t aspic .

# Запуск
docker run -d \
  --name aspic \
  -p 15191:15191 \
  -v $(pwd)/data:/opt/dix/aspic/data \
  aspic
```

Или через `docker-compose` (рекомендуется):

```yaml
version: '3.8'
services:
  aspic:
    build: .
    ports:
      - "15191:15191"
    volumes:
      - ./data:/opt/dix/aspic/data
    env_file:
      - .env
    restart: unless-stopped
```

### Напрямую

```bash
# 1. Клонировать репозиторий
git clone https://github.com/dneupokoev/aspic
cd aspic

# 2. Создать виртуальное окружение
python -m venv venv
source venv/bin/activate

# 3. Установить зависимости
pip install -r requirements.txt

# 4. Создать папки для данных
mkdir -p data/files data/preview

# 5. Настроить .env (скопировать из примера)
cp .env.example .env
# Отредактировать .env под свои нужды

# 6. Запустить
python -m app.main
```

## 📡 API

### 🔑 Основные эндпоинты (загрузка и просмотр)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/` | Главная страница - загрузка файлов |
| `POST` | `/api/preview` | Загрузить файл для предпросмотра |
| `POST` | `/api/confirm-upload` | Подтвердить загрузку (создать постоянную ссылку) |
| `GET` | `/view/{token}` | Страница просмотра файла с комментариями |
| `GET` | `/file/{token}` | Прямая ссылка на файл (для встраивания) |
| `GET` | `/download/{token}` | Скачать файл |

### 📊 Информационные эндпоинты

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/api/file/{token}` | Получить информацию о файле в JSON |
| `GET` | `/api/comments/{token}` | Получить комментарии к файлу |
| `GET` | `/api/captcha/{token}` | Получить капчу для удаления |
| `GET` | `/api/mime-map` | Получить маппинг MIME-типов |
| `GET` | `/api/upload-stats` | Получить статистику лимитов загрузок |

### ✏️ Действия с файлами

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/view/{token}/comment` | Добавить комментарий |
| `POST` | `/view/{token}/delete` | Удалить файл (с капчей и опциональным паролем) |

### 🚀 Прямая загрузка через API (`POST /api/upload`)

**Один запрос - файл загружен.** В отличие от двухшаговой схемы `preview` → `confirm-upload`, этот эндпоинт сохраняет файл сразу.

#### Авторизация

Требуется заголовок `X-Upload-Token` с валидным токеном. Токены настраиваются в `.env`:

```env
API_UPLOAD_TOKENS=tok_abc123:Менеджер Иван,tok_def456:API Сервис XYZ
```

Формат: `токен:Имя владельца`, через запятую. Пустое значение = эндпоинт отключён (503).

#### Запрос

```bash
curl -X POST http://localhost:15191/api/upload \
  -H "X-Upload-Token: tok_abc123" \
  -F "file=@/path/to/document.pdf" \
  -F "webhook_url=https://api.example.com/check-access" \
  -F "delete_password=MySecret123" \
  -F "expire_date=2026-06-01" \
  -F "ttl_hours=0"
```

#### Параметры (multipart/form-data)

| Поле | Тип | Обяз. | Описание |
|------|-----|:---:|----------|
| `file` | file | ✅ | Файл для загрузки |
| `X-Upload-Token` | header | ✅ | Токен авторизации |
| `webhook_url` | string | | URL вебхука для внешней авторизации |
| `delete_password` | string | | Пароль для удаления (4–16 символов) |
| `expire_date` | string | | Дата автоудаления (`YYYY-MM-DD`) |
| `ttl_hours` | int | | Срок жизни после последнего обращения (0 = вечно) |

#### Ответ (200 OK)

```json
{
  "status": "success",
  "token": "xY9kLmPq",
  "view_url": "/view/xY9kLmPq",
  "file_url": "/file/xY9kLmPq",
  "download_url": "/download/xY9kLmPq",
  "filename": "document.pdf",
  "size": 1234567,
  "mime_type": "application/pdf",
  "author": "Менеджер Иван"
}
```

#### Коды ошибок

| Статус | Описание |
|--------|----------|
| `401` | Неверный или отсутствующий токен |
| `400` | Ошибка валидации, размера файла или формата даты |
| `503` | Эндпоинт отключён (не настроены токены в `.env`) |
| `507` | Недостаточно места на диске |

#### Пример на Python (requests)

```python
import requests

url = "http://localhost:15191/api/upload"
headers = {"X-Upload-Token": "tok_abc123"}

with open("document.pdf", "rb") as f:
    files = {"file": f}
    data = {
        "expire_date": "2026-06-01",
        "delete_password": "MySecret123",
    }
    resp = requests.post(url, headers=headers, files=files, data=data)

print(resp.json())
# {'status': 'success', 'token': 'xY9kLmPq', 'view_url': '/view/xY9kLmPq', ...}
```

### 📤 Как загрузить файл через API

Загрузка состоит из **двух шагов**: сначала файл отправляется на предпросмотр, затем подтверждается создание ссылки.

#### Шаг 1: Предпросмотр (`POST /api/preview`)

Отправляем файл на сервер. Сервер возвращает `preview_id` и информацию о файле.

```bash
curl -X POST http://localhost:15191/api/preview \
  -F "file=@/path/to/document.pdf"
```

**Ответ:**
```json
{
  "preview_id": "abc123-def456-...",
  "preview_url": "/preview/abc123-def456-...",
  "filename": "document.pdf",
  "mime_type": "application/pdf",
  "size": 1234567,
  "size_formatted": "1.2 МБ",
  "icon": "📑",
  "type_name": "PDF"
}
```

#### Шаг 2: Подтверждение (`POST /api/confirm-upload`)

Создаём постоянную ссылку на файл.

```bash
curl -X POST http://localhost:15191/api/confirm-upload \
  -H "Content-Type: application/json" \
  -d '{
    "preview_id": "abc123-def456-...",
    "filename": "document.pdf",
    "mime_type": "application/pdf",
    "size": 1234567,
    "webhook_url": "",
    "delete_password": "",
    "expire_date": "2026-06-01",
    "ttl_hours": 0
  }'
```

**Ответ:**
```json
{
  "status": "success",
  "token": "xY9kLmPq",
  "file_url": "/view/xY9kLmPq",
  "preview_url": "/file/xY9kLmPq",
  "download_url": "/download/xY9kLmPq",
  "has_webhook": false,
  "expire_date": "2026-06-01 00:00:00",
  "ttl_hours": 0
}
```

#### Параметры confirm-upload

| Поле | Тип | Обязательное | Описание |
|------|-----|:---:|----------|
| `preview_id` | string | ✅ | ID из `/api/preview` |
| `filename` | string | ✅ | Имя файла |
| `mime_type` | string | ✅ | MIME-тип файла |
| `size` | int | ✅ | Размер файла в байтах |
| `webhook_url` | string | | URL вебхука для внешней авторизации |
| `delete_password` | string | | Пароль для удаления файла (4–16 символов) |
| `expire_date` | string | | Дата автоудаления в формате `YYYY-MM-DD` |
| `ttl_hours` | int | | Срок жизни после последнего обращения (0 = вечно) |
| `text_content` | string | | Для текстовых файлов - изменённое содержимое |

#### Загрузка больших файлов (до 4 ГБ)

Если в `.env` установлен `UNLIMITED_UPLOAD_SECRET`, сначала получите куку:

```bash
curl -c cookies.txt "http://localhost:15191/?upload_secret=ВАШ_СЕКРЕТ"
```

Затем используйте куку при загрузке:

```bash
curl -b cookies.txt -X POST http://localhost:15191/api/preview \
  -F "file=@/path/to/large_file.iso"
```

#### Ограничения для обычных пользователей

- Не более **100 файлов** за **6 часов** (скользящее окно)
- Не более **500 МБ** суммарно за окно
- Каждый файл до **100 МБ**
- При превышении - статус **429** с описанием и временем ожидания
