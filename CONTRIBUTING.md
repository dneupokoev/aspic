## 🚀 С чего начать

1. **Форкните репозиторий** на GitHub
2. **Склонируйте** свою копию:
   ```bash
   git clone https://github.com/dneupokoev/aspic.git
   cd aspic
   ```
3. **Создайте ветку** для вашей функции/исправления:
   ```bash
   git checkout -b feature/название-функции
   # или
   git checkout -b fix/название-исправления
   ```

## 🛠 Настройка окружения

### Требования:

- Python 3.10+
- pip
- Git

### Установка для разработки:

```bash
# Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows

# Установите зависимости
pip install -r requirements.txt
pip install -r requirements-dev.txt  # для разработки

# Скопируйте .env.example
cp .env.example .env

# Запустите в режиме разработки
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 15191
```

### Структура проекта:

```
aspic/
├── app/
│   ├── main.py          # Точки входа API
│   ├── database.py      # Работа с БД

## 📝 Стандарты кода

### Python:

- **PEP 8** — основной стиль кода
- **Type hints** — используйте аннотации типов
- **Docstrings** — документируйте функции и классы
- **Имена переменных** — `snake_case`
- **Имена классов** — `PascalCase`
- **Константы** — `UPPER_CASE`

Пример:
```python
def format_file_size(size: int) -> str:
    """Форматирует размер файла в человекочитаемый вид."""
    for unit in ['Б', 'КБ', 'МБ', 'ГБ']:
        if size < 1024:
            return f"{size:.1f} {unit}"
        size /= 1024
    return f"{size:.1f} ТБ"
```

### JavaScript:

- **ES6+** синтаксис
- **const/let** вместо var
- **camelCase** для переменных и функций
- **PascalCase** для классов

### HTML/CSS:

- **Семантические теги** (header, main, footer, section)
- **BEM** или подобная методология для CSS-классов
- **Мобильная адаптивность** — обязательно

### Комментарии:

- Пишите комментарии на **русском** (основной язык проекта)
- Объясняйте **почему**, а не **что**
- Удаляйте закомментированный код перед коммитом

## 🧪 Тестирование

### Запуск тестов:

```bash
# Запустить все тесты
pytest tests/

# Запустить с покрытием
pytest --cov=app tests/

# Запустить конкретный тест
pytest tests/test_webhook.py -v
```

### Требования к тестам:

- [ ] Новые функции должны иметь тесты
- [ ] Исправления багов должны иметь регрессионные тесты
- [ ] Покрытие кода не должно уменьшаться

### Типы тестов:

```
tests/
├── test_database.py     # Тесты БД
├── test_webhook.py      # Тесты вебхуков
├── test_upload.py       # Тесты загрузки
├── test_security.py     # Тесты безопасности
└── test_api.py          # Тесты API
```

## 📬 Pull Request

### Перед отправкой:

- [ ] Код соответствует стандартам
- [ ] Все тесты проходят
- [ ] Документация обновлена
- [ ] Нет предупреждений линтера
- [ ] Коммиты имеют понятные сообщения

### Шаблон PR:

```markdown
## Описание
Краткое описание изменений

## Тип изменений
- [ ] Исправление бага
- [ ] Новая функция
- [ ] Улучшение документации
- [ ] Рефакторинг
- [ ] Тесты

## Чек-лист
- [ ] Код отформатирован
- [ ] Тесты добавлены/обновлены
- [ ] Документация обновлена
- [ ] Нет breaking changes (или они обоснованы)

## Скриншоты (если применимо)
[Скриншоты изменений UI]

## Related Issues
Closes #123
```

### Процесс ревью:

1. **Автоматические проверки** (CI/CD)
2. **Ревью от мейнтейнера**
3. **Исправление замечаний** (если есть)
4. **Мерж** в основную ветку
